[Area]
1. Method 영역
- 클래스, 클래스(static) 변수, 멤버변수, 상수, 생성자, 메소드
- 클래스가 처음에 로딩될때 생성 ~ 프로그램이 종료가 될때 까지 메모리에 남아있음

2. Stack 영역
- 기본타입의 데이터, 지역변수(매개변수(파라미터) 포함)
- 메소드가 호출 ~ 메소드 종료

3. Heap 영역
- 참조타입 데이터 정보(객체를 가리키는 참조변수는 Stack), 인스턴스 변수, 배열
- new 객체 생성 ~ 객체 더이상 사용되지 않을때(*Garbage Collector), null값을 직접 할당할때까지



jvm은 일단 클래스의 정보를 컴파일러를 통해 읽고 그다음 불러옴

객체의 빈공간에는 null이 들어간다.

향상된 for문은 배열의 요소를 직접 수정하는것이 허용되지 않는다.
요소 수정을 위해서는 전통적인 for문을 사용해야함.
